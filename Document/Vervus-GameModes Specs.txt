GLiTCH! Specs
Appendix 1 — Exact Mode Specs
GLiTCH! family — standard, Blitz, and Chaos
Launch Game 1 — GLiTCH!
1.01 Concept
All players see an icon on their screen at the same time from a fixed set of recognizable shapes. Sometimes
everyone sees exactly the same icon; sometimes one player sees a relevant deviation. The group must very
quickly determine whether the round is a SYNC moment or a GLiTCH! moment and make the same
collective choice.
GLiTCH! does not rely on only one kind of difference. To keep the game deeper than a pure yes/no
pattern check, deviations may appear as:
• a clearly different icon,
• a near-match variant,
• or a a broken version of the same icon.
The core rule always remains the same:
• SYNC = all active players see the same relevant stimulus
• GLiTCH! = at least one active player sees a relevant deviation
1.02 10-second explanation
“Do all screens match? Press SYNC. If at least one screen is different, press GLiTCH!.”
1.03 Goal
Survive as long as possible as a team, build an increasingly higher combo, and keep answering correctly
while rounds become faster, more stressful, and less certain.
1.04 Core loop
• An icon is shown on all active screens at the same time
• The group quickly compares what they see
• The group decides collectively: SYNC or GLiTCH!
• All players press the same correct answer within the decision window
• On a correct answer, score, combo, and difficulty increase
• On the first mistake or missed input, LAST CHANCE is triggered
• On a failure during LAST CHANCE, the run ends immediately
1.04B Round cadence / transition between rounds
GLiTCH! must run as a short, rhythmic round cadence without an explicit between-round countdown.
GLiTCH! Specs
Each round consists of:
• icon reveal
• decision window
• result / lock phase
• transition beat
• next reveal
The result / lock phase starts as soon as:
• all players have entered input, or
• the decision window expires,
whichever happens first.
The next round starts automatically immediately after the transition beat.
Guideline timings:
• result / lock phase: ±500 ms
• transition beat: ±300 ms
Important intent:
• the game must feel fluid and rhythmic
• there must be just enough time for feedback and tension to land
• the flow must never stall in long pauses or annoying countdowns
• the transition must feel like a pulse, not like a brand-new round start
• the cadence must feel stable enough that players can trust the rhythm
1.05 Core mechanics
• simultaneous icon display
• one deviating screen or full sync (server-authoritative randomness)
• collective decision under time pressure
• combo / streak as the primary score layer
• increasing difficulty through timer pressure
• increasing GLiTCH! frequency in late game
• one shared LAST CHANCE per run
• direct fail-state after failed save or second mistake
GLiTCH! Specs
• reduced GLiTCH! frequency in the opening phase for onboarding
• multiple deviation types within the same core rule
1.05B Fixed icon set
GLiTCH! uses a fixed set of 5 icons reused across all rounds and all modes.
The icon set consists of:
• eye
• lightning bolt
• skull
• smiley
• star
1.05C Deviation types within GLiTCH!
GLiTCH! uses multiple deviation types inside the same rule system so the game does not flatten into a pure
binary shape-check.
Type A — Shape Swap
One player sees a clearly different icon from the others.
Examples:
• 3 players see a star, 1 player sees a skull
• 2 players see an eye, 1 player sees a lightning bolt
Purpose:
• extremely clear onboarding
• fast recognition
• low cognitive barrier
• immediate readability of the core GLiTCH! rule
Important rule:
• Shape Swap always uses only the 5 base icons from the fixed icon set
Type B — False Twin
One player does not see a completely different icon, but a near-match variant with a clear yet subtler
difference in shape or detail.
False Twin asset structure:
• each of the 5 icon families has 1 base icon and 2 False Twin variants
• the two False Twin variants are:
GLiTCH! Specs
 - Readable Twin
 - Doubt Twin
• total launch requirement: 10 False Twin variants across the 5 icon families
Examples:
• smiley with different mouth / eye layout
• eye with different pupil or outline shape
Purpose:
• increase social doubt
• force better comparison
• transition from obvious deviation into pressure-doubt
• deepen standard mode without adding extra rules
Important rule:
• False Twin deviations must remain intentionally readable
• the difference may be subtle, but never microscopic, arbitrary, or unreasonable
• Readable Twin is used earlier and more often than Doubt Twin
Type C — Partial Break
The icon remains the same recognizable family, but the shape appears broken, cracked, partially missing,
interrupted, or asymmetrically damaged.
Purpose:
• attack certainty rather than only shape recognition
• increase social uncertainty
Important rule:
• the deviation must be gameplay-relevant
• Partial Break must never be arbitrary
• the player must in principle be able to perceive and discuss the deviation
• the deviation must not collapse into random visual noise
1.05D General fairness rule for deviation types
All deviation types used in GLiTCH! must follow these principles:
• the relevant deviation must be in principle perceivable within the normal gameplay window
• doubt may arise from speed, comparison, and group pressure, not from unfair unreadability
GLiTCH! Specs
• subtle deviations must be intentionally designed, not accidental or sloppy
• Partial Break may attack confidence, but may not randomly destroy core information
• players should feel that they made a mistake, not that the system cheated
• any deviation type used in Blitz or Chaos must remain readable under that mode’s timing and presentation
pressure
1.06 Input actions
• SYNC button
• GLiTCH! button
• no input within the decision window = mistake
1.07 Information asymmetry
• In a SYNC round, everyone sees the same thing
• In a GLiTCH! round, one player sees a relevant deviation
• The deviating player does not automatically know that they are the deviator
• Tension comes from:
 - doubt
 - fast verbal comparison
 - consensus under pressure
 - confidence vs hesitation
1.08 Scaling 2–4 players
The rules remain identical for all group sizes.
• 2 players: faster route to consensus
• 3 players: first real majority / doubt dynamic
• 4 players: maximum pressure and discussion
Scaling factors:
• time pressure
• round tempo
• visual / audio intensity
• frequency of difficult late-game stress moments
GLiTCH! Specs
1.09 Win / loss
The run is fully team-based.
• Correct input by all players: +1 score / combo up
• One wrong input by one player: counts as a team mistake
• One missed input by one player: counts as a team mistake
• First team mistake: triggers LAST CHANCE
• Failure during LAST CHANCE: instant game over
• Second mistake later in the same run: instant game over
• On game over, the server records which player caused the decisive team mistake
1.09B LAST CHANCE mechanic
• Each run has one shared LAST CHANCE
• On the first mistake or missed input:
 - the game freezes very briefly (±500 ms)
 - visual + audio callout: “SAVE IT!”
• Immediately after that, a short panic replay of the exact same situation begins:
 - decision window: ±1.0 sec
 - same icons, same truth, no extra explanation
• On success:
 - the run continues
 - combo remains intact
• On failure:
 - instant game over
Important intent:
• LAST CHANCE must feel like a clutch panic-save
• LAST CHANCE is not a free extra turn
• the save must feel more stressful than the original round
GLiTCH! Specs

Launch Mode 2 — GLiTCH! Blitz
2.01 Concept
GLiTCH! Blitz uses the exact same core as standard GLiTCH!, but starts immediately on a late, stressful
section of the standard curve. The mode has no calm build-up and no LAST CHANCE. From the first second,
the group gets pure pressure, short reaction times, and immediate punishment for mistakes. On top of that
base curve, Blitz activates unpredictable Heat Surges: short peak moments where the next round lands
noticeably more aggressively.
2.02 10-second explanation
“Same rules as GLiTCH!, but full speed immediately. One mistake and it’s over. Sometimes the game spikes
harder.
2.03 Goal
GLiTCH! Specs
Score as high as possible in short, intense runs by reacting correctly from the first moment under heavy time
pressure, with no recovery mechanic, sudden spike moments layered on top, and occasional harder-toread rounds.
2.04 Base
Everything is identical to standard GLiTCH!, except for the deviations below.
2.05 Deviations from standard GLiTCH!
• no LAST CHANCE
• direct fail-state on first team mistake
• starts at late standard GLiTCH! intensity
• faster escalation curve
• extra end layer above the standard GLiTCH! ceiling
• temporary Heat Surges as an additional pressure layer
• tighter and more aggressive round cadence
• higher baseline audio / visual intensity
• same Shape Swap + False Twin structure as standard mode, but entered from a later pressure band
• limited Partial Break usage added to sharpen Blitz as a pro-facing mode without turning it into full Chaos
2.06 Win / loss
Deviation from standard GLiTCH!:
• first team mistake = instant game over
• there is no save-state or panic replay
2.07 Round cadence / transition between rounds
Blitz uses the same base structure as standard GLiTCH!, but tighter and more aggressive.
Guideline timings:
• result / lock phase: ±400 ms
• transition beat: ±350 ms
Important intent:
• the flow must never stall
• the game must feel like an aggressive pulse-run
• during a Heat Surge, the transition into the next round must feel noticeably more threatening than normal

2.09 Heat Surge logic
Heat Surges are the extra pressure layer in Blitz on top of the normal base curve.
Activation
• Heat Surges may only occur after the group has survived at least 2 correct rounds
• from the first valid round onward, each correct round has a server-side 20% activation chance
• after a Heat Surge, there is a cooldown of at least 2 rounds before another Heat Surge may occur
Effect
• one Heat Surge lasts 2 rounds
• during a Heat Surge, the underlying GLiTCH! chance stays equal to the normal base curve of that moment
• during a Heat Surge, the timer drops by an additional 0.3 sec temporarily
• during a Heat Surge, audio and visual intensity temporarily spike roughly 3 intensity levels above the
normal state
• after the Heat Surge ends, the mode returns to normal Blitz intensity
Transition into Heat Surge
When a Heat Surge activates, that must be announced explicitly and viscerally in the transition to the next
round.
GLiTCH! Specs
A Heat Surge transition includes:
• a noticeably heavier transition beat than normal
• an abrupt audiovisual warning pulse
• a short charge-up or surge callout of ±300 ms on top of the normal transition
• visibly brighter screen pulse / impact flash
• clearly more aggressive audio hit or rising pressure cue
Important intent:
• players must instantly feel that the next round is more dangerous
• the reaction should be: “oh shit”
• the warning must be short, but unmistakable
• the game may not stall in a long animation or cutscene
• the Heat Surge must feel like a sudden pressure explosion, not like an ordinary next round
Important intent:
• Heat Surges must feel like unexpected peak moments
• Heat Surges may surprise the group, but not unfairly overwhelm them
• Heat Surges are not fixed level steps, but temporary pressure explosions on top of the run

Launch Mode 3 — GLiTCH! Chaos
GLiTCH! Specs
3.01 Concept
GLiTCH! Chaos uses the exact same core as standard GLiTCH!, but lets the run gradually corrupt as combo
rises. The tension does not come from different input or different win / loss logic, but from a presentation
that becomes increasingly unstable, aggressive, and less trustworthy. The group is still playing the same
game, but it feels like the run itself is slowly falling apart.
3.02 10-second explanation
“Same rules as GLiTCH!, but the run starts falling apart, shapes begin to break, and everything becomes
harder to trust.”
3.03 Goal
Stay correct as a team for as long as possible while the run feels increasingly heavier and more unstable due
to rising visual and audio corruption.
3.04 Base
Everything is identical to standard GLiTCH!, except for the deviations below.
3.05 Deviations from standard GLiTCH!
• progressive visual corruption based on combo
• progressive audio distortion based on combo
• same base timing curve and LAST CHANCE logic as standard GLiTCH!
• higher subjective stress without different base rules
• different feedback layer and kill-screen vibe
• fairness rule explicitly dominant: presentation may sabotage confidence, but not break core information
• heavier use of uncertainty-driven deviation types
3.06 LAST CHANCE
Chaos uses the same LAST CHANCE logic as standard GLiTCH!.
Important intent:
• LAST CHANCE should feel more panicked in Chaos due to the existing instability
• The game may feel more intense during LAST CHANCE, but core information must stay fairly readable
• Chaos may make the save more stressful, but not unfairly sabotage it
3.07 Round cadence / transition between rounds
Chaos uses the same base structure as standard GLiTCH!, but the transition between rounds feels
increasingly unstable as combo rises.
Guideline timings:
GLiTCH! Specs
